EN:

Privacy policy: personal data that the application receives, processes or stores:

1. No personal data is stored outside the user's device, transferred anywhere or used. The application does not have permissions to use the Internet.
2. If the user has given permission to access contact data, then the list of Android user accounts is read and event data is obtained from the address book. The data is not stored anywhere, but only displayed in the application.
3. If the user has given permission to access calendar data, then the list of calendars and events in the selected calendars are read. The data is not stored anywhere, but only displayed in the application.
4. If the user specified any files to be used as event sources, then the events from these files are read. The data is not stored anywhere, but only displayed in the application.

Permissions that the application receives after installation or that may be requested during the application setup process:

Permission android.permission.READ_CONTACTS is required to obtain data from the Android address book. Information is not copied anywhere and does not change in any way.
Permission android.permission.WAKE_LOCK (prevent device from sleeping) is necessary to automatically update widgets on Android 8+.
Permission android.permission.RECEIVE_BOOT_COMPLETED (run at startup) is necessary to adjust the time of upcoming events notification.
Permission android.permission.GET_ACCOUNTS is required to obtain data about user accounts. Information is not copied anywhere and does not change in any way.
Permission android.permission.READ_EXTERNAL_STORAGE is necessary for playing notifications with a melody from external storage.
Permission android.permission.READ_CALENDAR is required to read calendar events. Information is not copied anywhere else and does not change in any way.
Permission android.permission.ACCESS_NOTIFICATION_POLICY is required to set high priority to quiz notifications on Android 7 and below.

Разрешение android.permission.READ_EXTERNAL_STORAGE необходимо для проигрывания уведомлений с мелодией из внешнего хранилища.
Разрешение android.permission.READ_CALENDAR необходимо для получения календарных событий. Информация больше никуда не копируется и никак не изменяется.
Разрешение android.permission.ACCESS_NOTIFICATION_POLICY необходимо для выставления высокого приоритета уведомлениям викторины на Android 7 и младше.

Die Berechtigung android.permission.WAKE_LOCK (Standby-Modus des Geräts verhindern) ist erforderlich, um Widgets unter Android 8+ automatisch zu aktualisieren.
Die Berechtigung android.permission.RECEIVE_BOOT_COMPLETED (wird beim Start ausgeführt) ist erforderlich, um den Zeitpunkt der Benachrichtigung über bevorstehende Ereignisse anzupassen.
Die Berechtigung android.permission.GET_ACCOUNTS ist erforderlich, um Android-Benutzerkontodaten abzurufen. Informationen werden nirgendwo anders kopiert und ändern sich in keiner Weise.
Die Berechtigung android.permission.READ_EXTERNAL_STORAGE ist erforderlich, um Benachrichtigungen mit einer Melodie aus einem externen Speicher abzuspielen.
Die Berechtigung android.permission.READ_CALENDAR ist erforderlich, um Kalenderereignisse zu empfangen. Informationen werden nirgendwo anders kopiert und ändern sich in keiner Weise.

隐私政策：应用程序接收、处理或存储的个人数据：

1. 个人数据不会存储在用户设备之外，也不会在任何地方传输或使用。 该应用程序没有使用 Internet 的权限。
2. 如果用户允许访问联系人数据，则读取 Android 用户帐户列表并从地址簿中获取事件数据。 数据不存储在任何地方，而仅显示在应用程序中。
3. 如果用户已授予访问日历数据的权限，则读取所选日历中的日历和事件列表。 数据不存储在任何地方，而仅显示在应用程序中。
4. 如果用户指定任何文件用作事件源，则从这些文件中读取事件。 数据不存储在任何地方，而仅显示在应用程序中。

应用程序在安装后收到的权限或在应用程序设置过程中可能请求的权限：

从Android通讯录获取数据需要android.permission.READ_CONTACTS和android.permission.GET_ACCOUNTS权限。 信息不会被复制到任何地方，并且不会以任何方式改变。
必须具有权限android.permission.WAKE_LOCK（防止设备进入休眠状态），才能在Android 8+上自动更新小部件。
权限android.permission.RECEIVE_BOOT_COMPLETED（在启动时运行）对于调整即将发生的事件的通知时间是必需的。
播放来自外部存储器的带有旋律的通知时，需要android.permission.READ_EXTERNAL_STORAGE权限。
接收日历事件需要android.permission.READ_CALENDAR。 信息不会复制到其他任何地方，并且不会以任何方式更改。
需要权限android.permission.ACCESS_NOTIFICATION_POLICY才能在Android 7及更低版本上为测验通知设置高优先级。
